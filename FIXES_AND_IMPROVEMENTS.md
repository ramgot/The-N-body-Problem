# Решённые проблемы и улучшения

## 🔴 Проблема 1: Ошибка "libsycl.so.8: cannot open shared object file"

### Причина
Intel oneAPI требует инициализации окружения для загрузки SYCL библиотек. Без активации скрипта `setvars.sh`, переменные окружения (особенно `LD_LIBRARY_PATH`) не устанавливаются.

### Решения (выбери один из трёх):

#### ✅ Способ 1: Wrapper скрипт (РЕКОМЕНДУЕТСЯ - самый простой)
```bash
./nbody_gpu.sh 1000 3600 86400
```
**Преимущества:**
- Не требует ручной активации
- Работает в любой сессии терминала
- Прозрачно для пользователя

#### ✅ Способ 2: Makefile target
```bash
make run-sycl
```
**Преимущества:**
- Удобно для скриптов
- Автоматически активирует oneAPI
- Консистентно с другими make целями

#### ✅ Способ 3: Постоянная активация
```bash
echo 'source /opt/intel/oneapi/setvars.sh > /dev/null 2>&1' >> ~/.bashrc
source ~/.bashrc
```
**Преимущества:**
- Работает для всех терминальных сессий
- Не требует wrapper скриптов
- Один раз и навсегда

## 🟢 Проблема 2: obj директория не создаётся образом

### Причина
Makefile не создавал директорию obj перед компиляцией.

### Решение
Добавлено явное правило в Makefile:
```makefile
obj:
	mkdir -p obj

setup: obj
	@mkdir -p $(OBJ_DIR) $(BUILD_DIR)
```
Теперь obj создаётся автоматически перед всеми целями.

## 📁 Созданные файлы

### 1. `nbody_gpu.sh` - Wrapper скрипт для GPU запуска
```bash
#!/bin/bash
source /opt/intel/oneapi/setvars.sh > /dev/null 2>&1
exec ./nbody_sycl "$@"
```
**Использование:** `./nbody_gpu.sh N DT T_MAX [SCENARIO]`

### 2. `nbody_omp.sh` - Wrapper для OpenMP (для симметрии)
Для удобства и консистентности с GPU wrapper.

### 3. `GPU_QUICKSTART.md` - Быстрый старт
Короткая инструкция с примерами для AMD Radeon 780M.

### 4. `test_gpu.sh` - Скрипт тестирования
Автоматически тестирует GPU vs CPU vs OpenMP и показывает сравнение производительности.

## 📌 Важные примечания

- Intel oneAPI 2025.0+ не поддерживает 32-bit библиотеки (не проблема для x64 сборок)
- Первый запуск GPU может быть медленнее (инициализация)
- Для N=1000+ GPU обычно быстрее CPU
- `test_gpu.sh` полезен для объективного сравнения производительности
